# 0.1.0 — 2026-06-14

Initial release of finnhub-mcp-server, wrapping the [Finnhub REST API](https://finnhub.io/docs/api) (v1). Fills the fleet's live-equity gap: current market prices and company financial context, starting from a company name rather than a ticker.

**API key required:** a free key from [finnhub.io/register](https://finnhub.io/register) must be set as `FINNHUB_API_KEY`. It is sent as the `token` query parameter and never appears in a tool input, log line, or thrown error. **Free tier scope:** US equities only, 60 req/min — international symbols (exchange-suffixed like `.TO`, `.DE`) return a clear `not_us_or_paid` error rather than a silent empty result.

## Added

- **`finnhub_search_symbols`** — resolve a company name or partial ticker to Finnhub symbols, US Common Stock first. Each result carries `isLikelyUS` (dot-suffix heuristic) so an agent can avoid spending a call on an international symbol the free tier cannot reach. `limit` (default 10) with truncation disclosure.
- **`finnhub_get_quote`** — real-time quote for one US symbol, paired with `/stock/market-status` so `priceIsLive` states whether `current` is a live price or the prior close. Detects the all-zero quote sentinel (`c=0, t=0, d=null, dp=null`) as `symbol_not_found`; carries `percentChange` (`dp`).
- **`finnhub_get_company`** — one-call company context combining `/stock/profile2` + `/stock/metric` + `/stock/peers`. Fundamentals are all-nullable and surfaced honestly (never zero-filled) for thinly-covered names; metrics/peers degrade to a `partial` array on sub-fetch failure while the profile drives errors.
- **`finnhub_get_earnings`** — two modes: `history` (a symbol's past quarters with actual-vs-estimate surprise %) and `calendar` (market-wide upcoming releases in a date window). `limit` (default 50) with truncation disclosure.
- **`finnhub_get_news`** — two modes: `company` (recent articles for one symbol over a date range) and `market` (headlines by category: general, forex, crypto, merger). `limit` (default 15) with truncation disclosure.
- **`finnhub_get_recommendations`** — analyst recommendation trends (strong-buy / buy / hold / sell / strong-sell counts per month, newest first). Empty array → `no_coverage`. `limit` (default 12) with truncation disclosure.
- **`finnhub://news-categories`** resource — the four valid market-news categories with one-line descriptions.

The API key rides in the `token` query parameter but is stripped from all thrown errors and log lines — only the endpoint path and operation name are surfaced. A bad or missing key fails loudly as a `ConfigurationError` at the first call, never masquerading as "no data".
